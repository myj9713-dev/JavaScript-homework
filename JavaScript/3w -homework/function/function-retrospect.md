# 1. 변수, 상수 작성 과제 수행 내용 및 회고

## 과제 수행 과정 기록
1. **데이터 성격 분류**
- 각 로직에서 사용되는 데이터를 분석하여 '변하는 값'과 '변하지 않는 값'으로 분류.
- 영화 기본가(basePrice)나 테스트 결과(result)처럼 연산 과정에서 업데이트가 필요한 항목은 `let`을 사용하고, 함수 자체의 정의나 기준값은 `const`로 선언하여 데이터의 성격을 명확히 했음.

2. **명명 규칙 수립**
- 협업 시 가독성을 높이기 위해 모든 함수와 변수명에 `camelCase`를 적용.

3. **가독성 최적화**
- 금액 데이터(14_000, 100_000) 작성 시 자바스크립트의 _(Numeric Separator) 기능을 활용하여 단위 파악을 쉽게 하였으며, 코드 테스트를 작성할 때는 console.log('\n---')를 삽입하여 콘솔 출력 결과가 시각적으로 분리되도록 구성.

4. **실행 및 검증**
 - 코드 작성 후 Quokka.js를 활용하여 `console.log`의 데이터 타입과 값이 정확히 할당되는지 확인.

---
## 테스트 코드 작성
각 로직이 의도대로 동작하는지 확인하기 위해 모든 변수를 `console.log()`로 직접 출력하여 검증.

1. **타입 검증**
- typeof 연산자를 활용하여 각 함수의 반환값이 string(메시지), number(비용/포인트), boolean(비밀번호 유효성) 등 요구사항에 맞는 데이터 타입으로 생성되는지 확인.

2. **변경 및 보호**
- 테스트를 위한 `result` 변수를 `let`으로 선언한 뒤, 여러 테스트 케이스의 결과값을 재할당 및 검증을 진행하며 효율적인 테스트 구조를 만듬.

3. **가독성 확인**
- 숫자 구분자(_)를 사용한 값이 실제 연산 시 누락 없이 숫자로 정상 인식됨을 확인하였고, 성공 시 ✅, 실패 시 ❌ 이모지를 활용해 가독성을 극대화.

## 과제 코드 리뷰
이번 과제에서는 총 5개의 핵심 비즈니스 로직을 구현하며 선언 키워드(let, const), 다양한 함수 선언 방식, 그리고 조건부 렌더링 로직을 준수하는 데 집중.

### **선언 키워드 및 함수 형태 선택**
1. **함수 다양성**
- function 선언문, 함수 표현식(const), 화살표 함수(=>)를 사용하여 자바스크립트의 다양한 함수 정의 방식을 학습.

2. **상수와 변수**
- 함수의 로직은 const로 보호하고, 계산 중간값인 basePrice는 let을 사용하여 상황에 따라 가격이 업데이트되도록 유연하게 설계.

### 명명 규칙 (Naming Convention)
1. **Camel Case**
- 모든 변수명에 카멜 케이스(예: `generateWelComeMessage`, `calculateDeliveryFee`)를 적용하여 단어 간 구분을 명확히 하고 가독성을 높임.

2. **Boolean Prefix**
-불리언 타입 변수에는 `is` 접두사(예: `isEarlyBird`)를 붙여 데이터의 타입을 이름만으로도 추측할 수 있게 설계.

### 논리 구조의 견고함
1. **복합 조건 연산**
- 배송비 계산 시 ||(OR) 연산자를, 비밀번호 검사 시 &&(AND) 연산자를 적재적소에 배치하여 복합적인 조건을 간결하게 표현.

2. **예외 처리(Default Return)**
- 포인트 적립이나 환영 메시지 로직에서 오타나 빈 값이 입력되더라도, 함수의 마지막 줄에 **기본값(Default)**을 반환하도록 설계하여 시스템의 중단을 방지.

### 데이터 표현의 가독성
**Numeric Separators**
- 큰 숫자 데이터(예: `27_200`, `42_000`)를 작성할 때 언더바(`_`)를 사용하여 수치의 정확성을 쉽게 파악할 수 있도록 처리.

## 회고(깨달은 점, 아쉬운 점 등)
변수와 상수의 과제처럼 변수명을 만드는게 제일 어려운 것 같다. 함수를 짤 때는 다행히 예습으로 if문을 공부한 적이 있어서 함수에 사용하는데 무리없이 사용할 수 있어서 좋았다. 그래도 AI의 도움을 받아 코드를 작성해가며 함수의 구조를 다시 한 번 공부하게 되는 시간이 된 것 같다. 
코드 테스트 작성은 처음 한번만 도움을 받고, 다른 문제의 코드 테스트 작성은 직접했는데, 일치 연산자를 사용해서 테스트를 진행하는게 신기했고, AI가 코드를 더 단축해서 사용하는 방법을 알려줬던 것도 있지만, 그것은 아직 내가 이해가 되지 않아서, 공부에 더 효과가 있도록 아직은 전부 풀어서 작성하는 것이 좋을 것 같다.
