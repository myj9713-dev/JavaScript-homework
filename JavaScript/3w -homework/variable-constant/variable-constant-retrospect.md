# 1. 변수, 상수 작성 과제 수행 내용 및 회고

## 과제 수행 과정 기록
1. **데이터 성격 분류**
- 13개의 항목을 '변하는 값'과 '변하지 않는 값'으로 분류. 
- 일시적인 정보(재고, 포인트)는 `let`, 설정값이나 물리값(URL, 빛의 속도)은 `const`로 정의.

2. **명명 규칙 수립**
- 협업 시 가독성을 높이기 위해 모든 변수명에 `camelCase`를 적용.
- 특히 예/아니오를 판단하는 데이터는 `isEmailVerified`처럼 질문형으로 이름을 지어 직관성을 높임.

3. **가독성 최적화**
- `MAX_UPLOAD_SIZE`와 같이 큰 숫자는 한눈에 들어오지 않아 자바스크립트의 `_`(Numeric Separator) 기능을 활용해 1,000단위로 구분하여 작성.

4. **실행 및 검증**
- 코드 작성 후 Quokka.js를 활용하여 `console.log`의 데이터 타입과 값이 정확히 할당되는지 확인.

## 테스트 코드 작성
각 로직이 의도대로 동작하는지 확인하기 위해 모든 변수를 `console.log()`로 직접 출력하여 검증.

1. **타입 검증**
- `typeof`를 통해 모든 변수가 요구사항(문자열, 숫자, 불리언)에 맞게 선언됨을 확인.

2. **변경 및 보호**
- `let` 변수는 재할당이 가능함을 확인하였고, `const` 상수는 재할당 시 에러가 발생하는 것을 확인하여 데이터 안정성을 검증함.

3. **가독성 확인**
- 큰 숫자에 `_`를 사용하여 가독성을 높인 부분이 실제 출력 시 정상적인 숫자로 인식됨을 확인.

## 과제 코드 리뷰
이번 과제에서는 총 13개의 변수와 상수를 선언하며, 데이터의 성격에 따라 적절한 선언 키워드(`let`, `const`)를 선택하고 명명 규칙을 준수하는 데 집중.

### 1. 선언 키워드 선택 (Declaration)
1. **변수(let)**
- 재고 수량, 조회수, 회원 포인트, 배송 상태와 같이 프로그램 실행 중 값이 수시로 변경될 가능성이 있는 항목은 `let`으로 선언하여 유연성을 확보.

2. **상수(const)**
- 빛의 속도, API URL, 최대 업로드 크기처럼 고정된 기준값이나 물리적 상수는 `const`를 사용하여 의도치 않은 값의 변경을 방지하고 코드의 안정성을 높임.

### 2. 명명 규칙 (Naming Convention)
1. **Camel Case**
- 모든 변수명에 카멜 케이스(예: `isEmailVerified`, `postViewCount`)를 적용하여 단어 간 구분을 명확히 하고 가독성을 높임.

2. **Boolean Prefix**
-불리언 타입 변수에는 `is` 접두사(예: `isCouponAvailable`, `isNotificationAgreed`)를 붙여 데이터의 타입을 이름만으로도 추측할 수 있게 설계.

3. **Constant Convention**
- 절대 변하지 않는 환경 설정값이나 물리 상수는 전체 대문자와 언더바(`_`)를 조합한 스네이크 케이스(예: `SPEED_OF_LIGHT`, `BASE_URL`)를 사용하여 변수와 시각적으로 차별화시킴.

### 3. 데이터 표현의 가독성
**Numeric Separators**
- 큰 숫자 데이터(예: `299_792_458`, `10_485_760`)를 작성할 때 언더바(`_`)를 사용하여 자릿수를 쉽게 파악할 수 있도록 처리.

## 회고(깨달은 점, 아쉬운 점 등)
변수명을 혼자서 해보려고 vs code에 있는 기능인 영어 변환 단축키로 확인했을때랑 AI의 도움을 받아 변수명을 설정할때랑 비교했을때, AI는 다른 개발자들이 사용하는 이름들을 추천해줘서 나도 보고 바로 이해가 갔지만, 영어 변환을 직접 코드에서 했을때는 너무 어려운 단어들만 추천이 있어서 변수명으로는 사용하기 어려워보였다.
이제 간단한 변수/상수 값을 할당하는 것이랑 그 선언된 내용들을 확인하는 단계는 이제 확실하게 기억할 수 있을 것 같다. 
비록 AI가 테스트 코드 작성하는 방법을 추천해줬을 때, 중괄호로 감싸서 한번에 테스트하거나, `${}`방식으로 추천을 해줬는데, 추천해준 코드를 어떻게 사용하는 건지 아직은 이해를 하지 못해서, 제일 기본 방식인 풀어서 하나하나 확인하는 방법으로 과제를 수행했다. 
나중에는 AI가 추천해준 방법을 내가 직접 이해하고 사용하는 단계까지 나아가기를 바란다.
