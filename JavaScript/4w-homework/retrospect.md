# 과제 수행 내용 및 회고
## 과제 수행 과정 기록
1. **데이터 성격 분류**
- 총 페이지 수(totalPages)와 카드 너비 등 변하지 않는 값은 const로, 페이지 번호처럼 변하는 값은 let으로 선언함.

2. **명명 규칙 수립**
- HTML 요소는 직관적으로 이해할 수 있도록 .card-list, .btn-prev, .current 등으로 명명함.

- 자바스크립트 함수는 화면을 갱신한다는 의미를 담아 updateCarousel로 정의하여 기능적 의미를 명확히 함.

3. **가독성 최적화**
- + 연산자를 활용한 문자열 연결 방식과 템플릿 리터럴(${}) 방식을 모두 검토하여, 팀원이나 협업자가 이해하기 쉬운 직관적인 코드 구조를 채택함.

- 중복되는 UI 업데이트 로직(텍스트 변경, 버튼 활성화/비활성화)을 하나의 함수에 모아 코드의 재사용성을 높임.

4. **실행 및 검증**
- CSS에서 키보드로 이동 시 파란색 보더 나오는 것을 검증함.

- 마우스 호버 했을 시 캐릭터 이미지가 확대되서 위로 이동한 것을 검증함.

- 텍스트 한 줄 넘었을 시 말줄임표 ... 로 표시되는 것을 확인.

---
## 테스트 코드 작성
- 각 로직이 의도대로 동작하는지 확인하기 위해 모든 변수를 `console.log()`로 직접 출력하여 검증.

- 페이지 전환 테스트: 버튼 클릭 시 currentIndex가 정상적으로 증가/감소하는지 콘솔로 확인.

---
## 과제 코드 리뷰
- 스타일 적용 방식의 선택: classList.add()와 .style.transform 방식을 모두 구현해 봄으로써 상황에 맞는 적절한 도구를 선택하는 법을 배움. 정해진 스타일은 클래스가 좋지만, 정교한 수치 계산이 필요한 캐러셀은 직접 스타일 조작이 유리함을 파악함.

- 레이아웃과 로직의 결합: CSS의 flex-wrap: nowrap 설정과 JS의 translateX 이동 수치가 일치해야만 UI가 깨지지 않는다는 점을 통해, 마크업 구조와 스크립트 설계의 밀접한 연관성을 이해함.

- 확장성 고려: 하드코딩된 숫자 대신 cardWidth와 gap 변수를 활용한 자동 계산 식을 도입하여, 향후 카드가 추가되거나 디자인이 변경되어도 JS 로직을 대대적으로 수정할 필요가 없도록 개선함.

---
## 회고(깨달은 점, 아쉬운 점 등)
솔직히 아직 CSS 지식이 많이 부족하다는 것을 더 깨닫게 되는 시간이었다. 마크업과 이것을 배치하는 CSS 디자인이 너무 어려웠다. AI의 도움으로 작성을 하고 내가 직접 수정을 하면서 모든 단계들에 대해 AI에게 질문을 하며 공부를 하게된 시간이었고, 이미 과제를 완성한 지금도 다시 0부터 시작하라고 하면 그래도 이미 작성된 내용이 있으니, 기억이 안날때마다 슬쩍슬쩍 보면서 작성은 가능할 것 같다. 금요일 수업이었던 실습을 base 폴더에서 혼자 진행 했을 때처럼 말이다!

캐러셀은 아직 배우지 않은 내용이라서 너무 낯설었지만, 이렇게 선행학습하고, 수업 시간때 다시 다루게 되면 더 잘 이해하지 않을까 싶다.

강사님이 생각하는 것을 AI에게 맡기지 말라고 하셨지만, 내 머리속에 지우개가 있어서...ㅠㅠ 아직은 열심히 질문을 하고 AI가 도와주는 것을 봐주셨으면 한다..
